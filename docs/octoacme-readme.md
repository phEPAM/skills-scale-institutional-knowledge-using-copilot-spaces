# OctoAcme Project Management Docs — README

## Welcome to OctoAcme Project Management

This directory contains the standardized processes, workflows, and guidance used by OctoAcme to plan, execute, track, and continuously improve projects across all cross-functional teams.

## Project Management Processes Overview

OctoAcme follows a structured, **iterative project lifecycle** grounded in these principles:

- **Customer-first**: Prioritize customer value and usability in every decision
- **Iterative delivery**: Deliver small, testable increments rather than big-bang releases
- **Clear ownership**: Each project has a named Project Manager and Product Lead
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback, learning, and blameless problem-solving

The lifecycle consists of five key phases: **Initiation**, **Planning**, **Execution**, **Release**, and **Retrospective & Continuous Improvement**.

## Key Artifacts Used Across Projects

- **Project Charter / One-pager** — Problem statement, goals, success metrics, stakeholders, timeline
- **Roadmap and Release Plan** — High-level delivery timeline and milestones
- **Sprint/Iteration Backlog** — Prioritized, estimated work items with acceptance criteria
- **Definition of Done** — Team's quality and acceptance standards
- **Risk Register** — Tracked risks with impact, likelihood, owners, and mitigations
- **Status Reports** — Weekly updates on progress, blockers, and decisions needed
- **Retrospective Notes** — Learnings and action items from sprints, releases, or incidents

## Communication Cadence

- **Weekly sync** between PM and Product Manager
- **Twice-weekly standups** for delivery team (or as agreed)
- **Monthly stakeholder updates** on overall progress
- **Ad-hoc escalations** for blockers and risks
